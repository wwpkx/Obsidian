- 在编译程序的时候使用‐g 选项生成调试信息

# 启动调试
```
- gdb <程序名> 
- gdb <程序名> core //程序非法执行后产生的 core dump 文件. 
- gdb <程序名> <pid> 指定进程号
- 读取符号表
	-symbols <file>
	-s
	-se
- core dump
	-core <file>
	-c <file>
- 源文件的搜索路径
	-directory <dir>
	-d <dir>
- 安静模式
	-q
```
# 例子
```
# gdb tools/system
# target remote:1234

```

|-----------------|-----------------------------------------|

| kill            | 停止程序执行。                                 |
| step            | 进入下一行代码的执行，会进入函数内部。                     |
| next            | 执行下一行代码。但不会进入函数内部。                      |
| until           | 继续运行直到到达指定行号，或者函数，地址等。                  |
| return          | 弹出选中的栈帧（stack frame）。如果后面指定参数，则返回表达式的值。 |
| stepi           | 执行下一条汇编/CPU指令。                          |
